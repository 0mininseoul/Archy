# 2026 재도전성공패키지 Gemini Gems 입력용 서비스 맥락 기획서

- 문서 버전: v1.0
- 작성일: 2026-03-01
- 대상 사업: 2026년도 재도전성공패키지 (중소벤처기업부 공고 제2026-87호)
- 작성 목적: Gemini Gems가 사업계획서(별첨1 양식) 초안을 정확한 구조로 생성하도록, Archy 서비스 맥락과 공고 제약을 한 문서에 통합

## 1) 공고/양식 핵심 제약 요약

### 1-1. 일정 및 운영
- 접수 기간: 2026-02-11 ~ 2026-03-04 16:00
- 협약 기간: 2026-04 ~ 2026-10 (7개월)
- 선정 절차: 서류평가 -> 요건검토(성실경영평가 포함) -> 발표평가 -> 최종선정
- 발표평가: 대표자 직접 발표 원칙, 20분 이내(발표 10분 + 질의응답)

### 1-2. 사업계획서 작성 제약
- 반드시 공고 첨부 양식(별첨1) 사용
- 목차 고정, 목차 변경/삭제 불가
- 본문 분량: 목차 페이지 제외 7페이지 이내 (붙임 제외)
- 양식 내 파란 안내 문구/가이드 표는 삭제 후 제출

### 1-3. 평가항목 및 배점
- 문제인식: 서류 35, 발표 25
- 실현가능성: 서류 30, 발표 35
- 성장전략: 서류 25, 발표 30
- 팀(기업) 구성: 서류 10, 발표 10
- 유의: 평가 단계별 60점 미만 시 탈락

### 1-4. 사업비 제약
- 정부지원사업비: 최대 1억원
- 총사업비 구성: 정부지원 75% 이하 + 자기부담 25% 이상
- 자기부담 구성: 현금 5% 이상 + 현물 20% 이하
- 사업계획서상 사업비와 시스템 입력 사업비가 다르면 사업계획서 기준으로 평가

## 2) Archy 서비스 정의 (심사용 핵심 문장)

Archy는 모바일 웹/PWA에서 음성 녹음만 하면, 20초 청크 단위 전사와 AI 문서 정리, Notion/Google Docs 저장, Slack/Push 알림까지 자동 처리하는 음성 기반 업무 문서화 서비스다.

## 3) 현재 제품 상태 (기술 실체)

### 3-1. 이미 구현된 핵심 기능
- 세션 기반 녹음 파이프라인: `start -> chunk -> finalize`
- 20초 청크 업로드, 실패 시 재시도 큐, 오프라인 복구
- STT: Groq Whisper Large V3
- 문서화: OpenAI `gpt-4o-mini` 기반 구조화 문서 생성
- Notion 페이지/DB 저장, Google Docs 자동 생성
- Slack 알림, Web Push 알림
- 사용자 맞춤 포맷 + 기본 포맷 지정
- 무료/프로 사용량 정책, 추천코드, 프로모션, 결제 연동

### 3-2. 비기능 특성
- 모바일 우선 UX(PWA)
- 인증/권한 기반 데이터 격리(Supabase)
- 오류 단계 추적(`processing_step`, `error_step`)
- 오디오 파일 기본 비저장 정책(사용자 토글 시만 저장)

## 5) 양식 항목별 작성 가이드 (별첨1 기준)

아래는 사업계획서 본문(7p)을 작성할 때 Gemini가 우선 반영해야 할 구조다.

### 5-1. 과제 개요
- 과제명: [K-Startup 입력 과제명과 동일하게 입력]
- 기업명: [예비재창업자면 "예비재창업자", 재창업기업이면 법인/개인 사업자명]
- 아이템 개요: "모바일 음성 녹음 기반 AI 자동 문서화 SaaS"
- 폐업 이력: [사용자 사실 입력 필수]

### 5-2. 1. 문제인식 (Problem)
#### 1-1. 폐업 원인 분석 및 개선 방안
- 핵심 메시지:
  - 이전 사업 실패 원인을 시장 타이밍/수익모델/운영체계 관점으로 구조화
  - 재창업에서는 제품-시장 검증과 반복 수익구조를 먼저 설계
- 필수 사용자 입력:
  - [폐업기업명/기간/아이템]
  - [실패 핵심원인 2~3개]
  - [이번 재창업에서의 구체 개선장치]

#### 1-2. 목표시장 고객 현황 및 필요성
- Archy 기본 논리:
  - 회의/강의/인터뷰 후처리 시간 낭비가 크고, 도구 분절로 전환비용이 큼
  - 모바일 현장에서 녹음 지속성/복구 안정성이 실제 문제
  - 전사 이후 정리/저장/공유까지 통합 자동화 니즈가 높음
- 필수 사용자 입력:
  - [초기 타깃 산업/직군]
  - [경쟁 대안 3개]
  - [타깃 고객 인터뷰/테스트 근거]

### 5-3. 2. 실현가능성 (Solution)
#### 2-1. 재창업 아이템 준비 현황
- 이미 갖춘 준비자산:
  - 서비스 아키텍처와 핵심 기능 구현 완료
  - 주요 외부연동(Notion/Google/Slack) 및 사용자 설정 체계 구축
  - 운영 문서(PRD/Feature/Flow) 정비
- 필수 사용자 입력:
  - [현재 사용자 수/활성도/테스트 수치]
  - [파일럿/PoC 사례]

#### 2-2. 실현 및 구체화 방안
- 개발/고도화 계획 예시:
  - 전사 품질 개선(도메인별 프롬프트/후처리)
  - 결과물 포맷 고도화(업종별 템플릿)
  - 연동 안정성 개선(재시도/에러복구)
  - 성능/비용 최적화(처리시간, API 비용)
- 필수 사용자 입력:
  - [7개월 내 기능 목표]
  - [정량 KPI: 정확도, 처리시간, 완료율]

### 5-4. 3. 성장전략 (Scale-up)
#### 3-1. 비즈니스 모델
- 기본 BM:
  - Free(월 사용량 한도) -> Pro(구독, 무제한) 전환
  - 팀/업무 협업 채널(Notion/Google/Slack) 중심 확산
- 필수 사용자 입력:
  - [가격/요금제 확정안]
  - [CAC, 예상 전환율, 유지율 가정]

#### 3-2. 사업화 추진 전략
- GTM 전략 뼈대:
  - 직군별 문제 중심 메시지(회의/리서치/교육)
  - 리퍼럴/콘텐츠/커뮤니티 유입
  - 파일럿 조직 확보 -> 사례 기반 확산
- 필수 사용자 입력:
  - [월별 고객획득 목표]
  - [세일즈/파트너 전략]

#### 3-3. 사업 추진 일정 및 자금 운용
- 협약기간(2026.04~10) 기준으로 월별 목표를 설정
- 사업비는 비목 정의(재료비/외주용역비/기계장치/인건비/광고선전비 등)에 맞춰 산출근거 명시

### 5-5. 4. 기업 구성 (Team)
#### 4-1. 조직 구성 및 보유역량
- 기술 구현 역량, 제품 운영 역량, 사업개발 역량을 역할별로 분리해 제시
- 대표자 중심 의사결정/실행 체계 명확화

#### 4-2. 조직 구성 계획
- 협약기간 내 채용/외부전문가 활용 계획 제시
- 개발, 마케팅, 고객성공, 운영관리 직무의 우선순위 명확화
- 필수 사용자 입력:
  - [현재 인력수(대표 포함)]
  - [추가 채용 인원/시점/직무]

## 7) 사업비 설계 가이드 (작성용)

### 7-1. 제약 체크
- 정부지원사업비 <= 100,000,000원
- 정부지원사업비 / 총사업비 <= 75%
- 자기부담 현금 / 총사업비 >= 5%
- 자기부담 현물 / 총사업비 <= 20%

### 7-2. 예시(정부지원 1억원 신청 시)
- 총사업비: 133,333,333원
- 정부지원: 100,000,000원
- 자기부담 현금: 6,666,667원 이상
- 자기부담 현물: 26,666,666원 이하

### 7-3. 비목 배분 예시
- 인건비: 핵심개발/제품 운영
- 외주용역비: UI/디자인, QA, 보안 점검 등
- 기계장치(SW 포함): 분석/운영 도구
- 광고선전비: 랜딩, 콘텐츠, 퍼포먼스 테스트
- 지급수수료: 인증/특허/전시/멘토링 등

## 8) Gemini Gems 사용 지시문 (복붙용)

아래 텍스트를 Gems 프롬프트 시작부에 넣는다.

```text
너는 2026 재도전성공패키지 사업계획서 전문 코치다.
입력 문서를 기반으로 별첨1 양식(1-1, 1-2, 2-1, 2-2, 3-1, 3-2, 3-3, 4-1, 4-2)에 맞춰 한국어 사업계획서를 작성하라.
반드시 다음을 지켜라.
1) 사실이 없는 내용은 창작하지 말고 [추가 입력 필요]로 표시
2) 평가배점(문제인식/실현가능성/성장전략/팀)에 맞게 분량을 차등 배분
3) 협약기간은 2026.04~2026.10(7개월)로 고정
4) 사업비는 정부지원 75% 이하, 현금 5% 이상, 현물 20% 이하 제약을 유지
5) 문장은 심사위원 관점에서 구체성과 실행가능성을 우선
결과물 형식:
- A안: 서류평가용 7페이지 분량 본문 초안
- B안: 발표평가용 10분 발표 스크립트(핵심 논리만)
- C안: 추가로 필요한 입력정보 체크리스트
```
